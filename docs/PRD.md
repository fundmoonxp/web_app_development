# PRD 文件：簡易記帳系統

**專案名稱**：簡易記帳系統
**專案描述**：為大學生與小資族設計的網頁記帳系統，透過快速記帳、預算提醒與圖表分析，幫助用戶輕鬆掌握財務狀況與建立良好理財習慣。

## 1. 專案概述
- **背景與動機**：大學生與小資族常因日常消費零碎而難以追蹤金錢流向，導致月底常有透支或不知錢花去哪的困擾。過於複雜的現有系統容易降低用戶每日記帳的意願，亟需一個直覺且功能恰到好處的輕量級方案。
- **目標用戶**：大學生、初入社會的小資族
- **核心價值主張**：以簡單直覺的操作介面，提供快速記帳與視覺化圖表分析，加上預算與多帳戶管理，協助用戶輕鬆且精準地建立財務目標。

## 2. 功能需求

### 2.1 快速記帳與自定義分類
- 作為 **記帳用戶**，我希望 **能在一頁內快速輸入金額與類別**，以便 **在消費後能立刻記錄，不浪費時間**。
- 作為 **記帳用戶**，我希望 **能新增、修改與刪除自定義的收支分類**，以便 **根據我的個人習慣紀錄不同種類的開銷**。

### 2.2 預算上限提醒
- 作為 **需要控管開銷的用戶**，我希望 **能設定每個月或每個分類的預算上限**，以便 **在花費接近或超過設定額度時獲得系統的進度條警示或提醒，避免超支**。

### 2.3 固定支出自動帶入
- 作為 **有固定開銷（如房租、電信費、影音訂閱費）的用戶**，我希望 **系統能設定定期自動紀錄這些款項**，以便 **省去每月手動重複輸入的時間並避免遺漏**。

### 2.4 消費視覺化分析圖表
- 作為 **關心財務狀況的用戶**，我希望 **能在儀表板看到圓餅圖或長條圖顯示當月/當週的支出結構**，以便 **快速抓出最大的開銷來源在哪裡，進而調整消費**。

### 2.5 多帳戶/支付工具管理
- 作為 **使用多種支付方式的用戶**，我希望 **能管理不同帳戶（如現金、信用卡、銀行帳戶、電子支付）的各別餘額與流水帳**，以便 **清楚對應各個帳戶的實際可用資金並核對帳務**。

## 3. 非功能需求
- **技術限制**：
  - 後端全面採用 **Flask** 框架。
  - 前端使用 **HTML/CSS/JavaScript + Jinja2** 模板。
  - 資料庫使用輕量原生的 **SQLite**。
- **效能考量**：操作如新增紀錄或切換報表頁面的載入時間須小於 1 秒，確保操作流暢性。
- **安全考量**：
  - 用戶密碼必須經過單向雜湊加密（如 bcrypt）。
  - 對用戶輸入進行完整校驗，防範 SQL Injection 與 XSS。

## 4. MVP 範圍

### Must Have (核心必備)
- 基本的加減記帳（紀錄收支金額、時間、分類與備註）
- 總收入、總支出與結餘的計算與顯示
- 內建常見的記帳分類（食衣住行育樂）

### Should Have (重要優先)
- 多帳戶/支付工具管理
- 消費視覺化分析圖表（分類圓餅圖）
- 預算上限進度條顯示與警告

### Nice to Have (有會更好)
- 自定義記帳分類
- 固定支出自動帶入
- 帳目資料匯出成 CSV
